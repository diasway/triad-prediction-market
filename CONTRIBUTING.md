# Contribution Rules

## Commit message convention

Use Conventional Commits:

- `feat(scope): message`
- `fix(scope): message`
- `test(scope): message`
- `docs(scope): message`
- `refactor(scope): message`
- `chore(scope): message`
- `ci(scope): message`

Bad examples: `final`, `fixed stuff`, `asdf`.

## Pull request checklist

- Forge build passes.
- Full test suite passes.
- Coverage is updated.
- Slither has zero High and zero Medium findings.
- Frontend build passes.
- Documentation is updated.
